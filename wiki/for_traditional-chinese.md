# Tcl 中的 "for" 循環語句：用於迴圈控制的強大工具

## 概要
在 Tcl 編程語言中，`for` 循環是一種控制結構，用於重複執行一段程式碼，直到指定的條件不再滿足。這使得處理重複任務和迭代數據變得簡單而高效。

## 文檔
### 目的
`for` 循環的主要目的是讓開發者能夠執行一段代碼多次，根據初始化值、條件和更新步驟來控制迴圈的執行次數。

### 使用方式
`for` 循環的基本語法如下：

```tcl
for {初始化} {條件} {更新} {
    # 迴圈內的程式碼
}
```

- **初始化**：在迴圈開始前執行的代碼，通常用來設置迴圈變量的初始值。
- **條件**：每次迴圈開始前評估的表達式，當這個表達式為假時，迴圈停止。
- **更新**：每次迴圈結束後執行的代碼，通常用來改變迴圈變量的值。

### 詳細說明
在 Tcl 中，`for` 循環可以執行任何合法的 Tcl 程式碼，並且可以嵌套使用。注意，`for` 循環中的每一部分都需要用大括號 `{}` 包圍，這樣才能正確解析 Tcl 代碼。

## 範例
以下是 `for` 循環的基本使用範例：

### 範例 1：簡單的數字迴圈
```tcl
for {set i 0} {$i < 5} {incr i} {
    puts "目前的值是：$i"
}
```
這段代碼將輸出：
```
目前的值是：0
目前的值是：1
目前的值是：2
目前的值是：3
目前的值是：4
```

### 範例 2：計算總和
```tcl
set sum 0
for {set i 1} {$i <= 10} {incr i} {
    set sum [expr {$sum + $i}]
}
puts "1 到 10 的總和是：$sum"
```
這段代碼將輸出：
```
1 到 10 的總和是：55
```

## 解釋
在使用 `for` 循環時，開發者應注意以下幾點：
- 確保條件表達式能夠在某個時候變為假，否則會導致無窮迴圈。
- 初始化、條件和更新部分必須用大括號包圍，以便 Tcl 正確解析它們。
- 可以在迴圈內部使用其他 Tcl 控制結構，如 `if`、`while` 等。

## 一句總結
Tcl 的 `for` 循環是一個靈活的迴圈控制結構，能夠高效地執行重複任務。
