# التعامل مع الملفات في Tcl: دليلك الشامل

## ملخص
تعتبر أوامر إدارة الملفات في Tcl من الأدوات الأساسية التي تسهل التعامل مع الملفات والمجلدات، مما يتيح لك القراءة، الكتابة، والحذف، وغيرها من العمليات على الملفات بكل سهولة.

## الوثائق
تقدم Tcl مجموعة متنوعة من الأوامر للتعامل مع الملفات. تشمل هذه الأوامر:

- **file**: يُستخدم لإنشاء، قراءة، وكتابة الملفات. يمكنك استخدامه أيضًا لإدارة المجلدات.

### الاستخدام
يستخدم الأمر `file` مع مجموعة متنوعة من الخيارات، مثل:

- `file exists <path>`: يتحقق مما إذا كان الملف أو المجلد موجوداً.
- `file delete <path>`: يحذف ملفاً أو مجلداً.
- `file copy <source> <destination>`: ينسخ ملفاً من مسار إلى آخر.
- `file rename <oldName> <newName>`: يعيد تسمية ملف.
- `file permissions <path>`: يستعرض أو يعدل أذونات الوصول إلى ملف أو مجلد.

### التفاصيل
يمكن استخدام الأمر `file` مع مسارات مطلقة أو نسبية. يُفضل استخدام المسارات المطلقة لتجنب أي لبس. عند العمل مع الملفات، تأكد من وجود الأذونات اللازمة للقيام بالعمليات المطلوبة.

## أمثلة

### مثال 1: التحقق من وجود ملف
```tcl
if {[file exists "example.txt"]} {
    puts "الملف موجود."
} else {
    puts "الملف غير موجود."
}
```

### مثال 2: حذف ملف
```tcl
file delete "old_file.txt"
puts "تم حذف الملف."
```

### مثال 3: نسخ ملف
```tcl
file copy "source.txt" "destination.txt"
puts "تم نسخ الملف بنجاح."
```

### مثال 4: إعادة تسمية ملف
```tcl
file rename "old_name.txt" "new_name.txt"
puts "تمت إعادة التسمية."
```

## الشرح
عند استخدام الأوامر الخاصة بالملفات، تأكد من التعامل مع الأخطاء بشكل صحيح. على سبيل المثال، قد تواجه مشاكل في الأذونات عند محاولة حذف أو تعديل ملفات معينة. من المهم أيضاً الانتباه إلى المسارات، لأن استخدام مسار غير صحيح يمكن أن يؤدي إلى أخطاء في القراءة أو الكتابة.

## ملخص بجملة واحدة
تعتبر أوامر `file` في Tcl أدوات فعالة لإدارة الملفات والمجلدات، مما يسهل عليك إجراء عمليات متعددة على الملفات بكل سهولة.
