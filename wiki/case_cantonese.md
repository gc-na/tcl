# Tcl 中的 case 命令：具條件的選擇結構

## 概述
`case` 命令係 Tcl 編程語言中一個強大嘅條件選擇結構，允許開發者根據一個表達式嘅值來執行不同嘅代碼塊。佢適合用於需要根據多個可能選項進行判斷嘅場景。

## 文檔
### 目的
`case` 命令用於根據特定條件選擇執行哪一段代碼，提供了一種清晰嘅方式來處理多個選擇。

### 用法
基本語法如下：
```tcl
case value {
    pattern1 {
        # 執行代碼塊1
    }
    pattern2 {
        # 執行代碼塊2
    }
    ...
    default {
        # 執行默認代碼塊
    }
}
```

- **value**：一個表達式，會用來同後面嘅模式進行匹配。
- **pattern**：可以係一個具體值或者使用通配符（例如 `*`）來匹配多個值。

### 詳情
- `case` 命令會依次檢查每個模式，並執行第一個匹配到的代碼塊。
- 如果冇任何模式匹配，則執行 `default` 部分（如有提供）。

## 示例
### 基本用法
```tcl
set day "星期一"

case $day {
    "星期一" {
        puts "今日係星期一！"
    }
    "星期二" {
        puts "今日係星期二！"
    }
    "星期三" {
        puts "今日係星期三！"
    }
    default {
        puts "今個星期日子未定！"
    }
}
```

### 使用通配符
```tcl
set animal "貓"

case $animal {
    "狗" {
        puts "你有一隻狗！"
    }
    "貓" {
        puts "你有一隻貓！"
    }
    "*兔*" {
        puts "你有一隻兔！"
    }
    default {
        puts "你有其他動物！"
    }
}
```

## 解釋
- **常見陷阱**：確保模式之間冇重疊，因為佢會按順序檢查，第一個匹配就會執行，後面嘅模式將不會被檢查。
- **匹配問題**：使用通配符時要小心，確保你嘅模式正確，以避免意外匹配。
- **代碼塊結束**：每個代碼塊必須以大括號 `{}` 包圍，以便 Tcl 正確解析。

## 單行摘要
`case` 命令係 Tcl 中用來根據一個表達式的值選擇執行代碼塊的有效工具。
