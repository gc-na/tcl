# Tcl의 "global" 명령어: 전역 변수 접근 방법

## 개요
Tcl의 "global" 명령어는 전역 변수를 사용하여 스코프를 넘나드는 데이터를 다룰 수 있도록 해줍니다. 이 명령어를 통해 프로시저 안에서 전역 변수를 선언하고 사용할 수 있습니다.

## 문서화
Tcl에서 `global` 명령어는 특정 프로시저 내에서 전역 변수를 참조하거나 수정할 수 있도록 허용합니다. 기본적으로 Tcl에서 변수는 지역 스코프를 가지지만, `global` 키워드를 사용하면 전역 변수에 접근할 수 있습니다.

### 용도
전역 변수를 사용해야 할 상황은 주로 여러 프로시저 간에 데이터를 공유할 필요가 있을 때 발생합니다. `global` 명령어를 사용하면 이러한 변수에 접근할 수 있으며, 이를 통해 프로그램의 다양한 부분에서 동일한 데이터를 사용할 수 있습니다.

### 사용법
```tcl
global varName1 varName2 ...
```
여기서 `varName1`, `varName2`는 전역 변수의 이름입니다. 이 명령어는 프로시저 내에서 해당 전역 변수들을 사용할 수 있게 만듭니다.

## 예제
### 기본 사용 예제
```tcl
set x 10  ;# 전역 변수 x 설정

proc incrementX {} {
    global x  ;# 전역 변수 x 접근
    set x [expr {$x + 1}]  ;# x 값 증가
}

incrementX
puts $x  ;# 출력: 11
```

### 복수 변수 사용 예제
```tcl
set a 5
set b 10

proc add {} {
    global a b  ;# 전역 변수 a, b 접근
    return [expr {$a + $b}]  ;# a와 b의 합 반환
}

puts [add]  ;# 출력: 15
```

## 설명
- **일반적인 실수**: `global` 명령어를 사용하지 않고 전역 변수에 접근하려고 하면 Tcl은 해당 변수를 찾지 못해 에러를 발생시킵니다. 따라서 전역 변수를 사용하는 모든 프로시저에는 반드시 `global` 키워드를 추가해야 합니다.
- **이름 충돌**: 지역 변수와 전역 변수의 이름이 동일할 경우, 프로시저 내에서는 지역 변수가 우선시됩니다. 이로 인해 예상치 못한 동작을 초래할 수 있으므로 변수 이름 관리에 주의해야 합니다.
- **성능 고려**: 너무 많은 전역 변수를 사용하는 것은 코드의 가독성을 떨어뜨릴 수 있으며, 유지보수를 어렵게 만들 수 있습니다. 필요 최소한의 전역 변수만 사용하는 것이 좋습니다.

## 한 줄 요약
Tcl의 `global` 명령어는 프로시저 내에서 전역 변수에 접근하고 수정할 수 있게 해주는 중요한 기능입니다.
