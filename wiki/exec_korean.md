# Tcl exec 명령어: 외부 명령 실행하기

## 개요
Tcl의 `exec` 명령어는 외부 프로세스를 실행하고 그 결과를 Tcl 스크립트로 가져오는 데 사용됩니다. 이 명령어는 시스템 명령어 또는 프로그램을 호출하여 그 출력을 Tcl에서 사용할 수 있도록 합니다.

## 문서화
`exec` 명령어는 운영 체제의 셸에서 명령을 실행하고 결과를 Tcl 리스트로 반환하는 기능을 제공합니다. 이 명령어는 다양한 파라미터를 지원하며, 다음과 같은 형태로 사용됩니다:

```tcl
exec command ?arg arg ...?
```

### 목적
`exec`는 외부 명령을 호출하여 그 결과를 Tcl로 가져오고, 이를 통해 스크립트에서 시스템 리소스를 활용할 수 있게 해줍니다.

### 사용법
- `command`: 실행할 외부 명령어를 지정합니다.
- `arg`: 명령어에 전달할 추가 인자입니다. 여러 개의 인자를 공백으로 구분하여 지정할 수 있습니다.

### 세부 사항
- `exec`는 명령어의 실행이 완료될 때까지 Tcl 스크립트를 차단합니다.
- 출력은 명령어가 표준 출력으로 반환한 내용을 기반으로 하며, 표준 오류는 Tcl의 오류로 처리됩니다.
- `exec`는 실패할 경우 Tcl 오류를 발생시킵니다.

## 예제
1. 간단한 파일 목록 보기:
   ```tcl
   set files [exec ls]
   puts "Files: $files"
   ```

2. 시스템 날짜 출력하기:
   ```tcl
   set currentDate [exec date]
   puts "Current date: $currentDate"
   ```

3. 파라미터 전달하기:
   ```tcl
   set result [exec echo "Hello, World!"]
   puts $result
   ```

## 설명
- `exec`를 사용할 때, 명령어가 존재하지 않거나 실행 권한이 없으면 Tcl이 오류를 발생시킵니다.
- 외부 명령어가 예상치 못한 오류를 반환할 수 있으므로, 항상 오류 처리를 고려해야 합니다.
- Windows와 Unix 계열 운영 체제에서 명령어의 형식이나 작동 방식이 다를 수 있으므로, 플랫폼에 따라 테스트가 필요합니다.
