# Tcl의 "load" 명령어: 동적 라이브러리 로딩

## 개요
Tcl에서 "load" 명령어는 동적 라이브러리를 로딩하여 Tcl 스크립트에서 사용할 수 있도록 하는 기능을 제공합니다. 이 명령어를 사용하면 C로 작성된 확장 모듈을 Tcl 인터프리터에서 사용할 수 있게 됩니다. 

## 문서
### 목적
"load" 명령어는 Tcl 스크립트에서 외부 C 라이브러리를 동적으로 로드하여 해당 라이브러리의 함수를 Tcl에서 호출할 수 있게 합니다. 이를 통해 개발자는 C로 작성된 성능이 중요한 기능을 Tcl의 스크립트 환경에서 활용할 수 있습니다.

### 사용법
```tcl
load <라이브러리 이름> ?<초기화 인자>?
```
- `<라이브러리 이름>`: 로드할 동적 라이브러리의 경로 또는 이름.
- `<초기화 인자>`: 선택적으로 초기화에 필요한 인자를 지정할 수 있습니다.

### 상세 설명
- "load" 명령어는 Tcl의 확장 기능을 사용할 수 있도록 해줍니다. 로드된 라이브러리는 Tcl 명령어로 사용할 수 있는 함수와 변수를 제공합니다.
- 해당 라이브러리는 반드시 시스템에 설치되어 있어야 하며, Tcl 인터프리터가 해당 라이브러리를 찾을 수 있는 경로에 위치해야 합니다.
- 라이브러리가 성공적으로 로드되면, Tcl 스크립트에서 해당 라이브러리에 정의된 명령어를 사용할 수 있습니다.

## 예제
### 기본 사용 예
```tcl
# 예: mathlib라는 라이브러리 로드
load ./mathlib.so

# 로드된 라이브러리의 함수 호출
set result [math_add 5 10]
puts "결과는: $result"
```

### 초기화 인자 사용 예
```tcl
# 예: 초기화 인자를 사용하는 경우
load ./myextension.so "초기화 인자"
```

## 설명
- **경로 문제**: "load" 명령어는 라이브러리의 경로를 정확하게 지정해야 하며, 파일이 존재하지 않거나 경로가 잘못된 경우 오류가 발생합니다.
- **라이브러리 의존성**: 로드하려는 라이브러리가 다른 라이브러리에 의존하는 경우, 모든 의존 라이브러리도 적절히 로드되어야 합니다.
- **플랫폼 차이**: Windows, macOS, Linux 등 운영체제에 따라 동적 라이브러리의 확장자가 다르므로, 이를 유의해야 합니다. 예를 들어, Windows에서는 `.dll`, Linux에서는 `.so`, macOS에서는 `.dylib`를 사용합니다.

## 한 문장 요약
Tcl의 "load" 명령어는 외부 동적 라이브러리를 로드하여 Tcl 스크립트에서 해당 라이브러리의 기능을 사용할 수 있도록 해줍니다.
