# Tcl 파일 처리 명령어: file

## 개요
Tcl의 `file` 명령어는 파일 시스템과 상호작용하기 위한 다양한 기능을 제공하여 파일과 디렉토리를 생성, 삭제, 이동, 정보 조회 등을 가능하게 합니다. 이 명령어는 파일 작업을 간편하게 수행할 수 있도록 돕습니다.

## 문서
`file` 명령어는 Tcl 스크립트에서 파일과 디렉토리를 관리하는 데 사용됩니다. 이 명령어는 파일 경로의 조작, 파일 존재 여부 확인, 파일 속성 조회 등 여러 기능을 지원합니다.

### 사용법
`file` 명령어는 다음과 같은 서브커맨드를 통해 다양한 작업을 수행할 수 있습니다:

- `file exists <경로>`: 지정된 경로에 파일이나 디렉토리가 존재하는지 확인합니다.
- `file size <경로>`: 지정된 파일의 크기를 바이트 단위로 반환합니다.
- `file type <경로>`: 지정된 파일의 유형을 확인합니다.
- `file delete <경로>`: 지정된 파일이나 디렉토리를 삭제합니다.
- `file copy <원본 경로> <대상 경로>`: 원본 파일을 대상 경로로 복사합니다.
- `file rename <원래 경로> <새로운 경로>`: 파일이나 디렉토리의 이름을 변경합니다.

## 예제
```tcl
# 파일 존재 여부 확인
if {[file exists "example.txt"]} {
    puts "파일이 존재합니다."
} else {
    puts "파일이 존재하지 않습니다."
}

# 파일 크기 조회
set size [file size "example.txt"]
puts "파일 크기: $size 바이트"

# 파일 복사
file copy "example.txt" "copy_of_example.txt"
puts "파일 복사가 완료되었습니다."

# 파일 삭제
file delete "copy_of_example.txt"
puts "파일 삭제가 완료되었습니다."
```

## 설명
`file` 명령어를 사용할 때 주의해야 할 점은 다음과 같습니다:

- **경로 구문**: Tcl은 경로 구문에서 슬래시(`/`)와 백슬래시(`\`)를 모두 지원하지만, 플랫폼에 따라 다르게 해석될 수 있습니다. 가능한 경우 슬래시를 사용하는 것이 좋습니다.
- **파일 시스템 권한**: 파일이나 디렉토리를 삭제하거나 변경하려고 할 때, 권한이 없으면 오류가 발생할 수 있습니다. 따라서 필요한 권한을 확인해야 합니다.
- **비어 있는 디렉토리 삭제**: `file delete` 명령어는 비어 있지 않은 디렉토리를 삭제할 수 없습니다. 이 경우, 먼저 디렉토리 내의 모든 파일을 삭제해야 합니다.
