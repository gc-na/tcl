# Tcl의 "break" 명령어: 사용법과 예제

## 개요
Tcl에서 `break` 명령어는 루프나 프로시저 내에서 흐름 제어를 위해 사용되며, 현재의 반복 또는 프로시저를 중단하는 데 사용됩니다.

## 문서화
### 목적
`break` 명령어는 Tcl 스크립트에서 반복문 또는 프로시저 실행을 중단하고, 다음 단계로의 진행을 막는 용도로 사용됩니다.

### 사용법
`break`는 반복문(예: `foreach`, `for`, `while`)이나 프로시저 내에서 호출될 수 있습니다. 사용자가 특정 조건을 만족했을 때 반복을 중단하고자 할 때 유용합니다.

#### 구문
```tcl
break
```

### 세부 사항
- `break`는 현재 실행 중인 루프를 즉시 종료하고, 루프에 의해 호출된 다음 명령어로 제어를 이동합니다.
- 프로시저 내에서 `break`를 호출할 경우, 해당 프로시저의 실행도 종료됩니다.
- `break`는 루프의 중첩 상태에서 가장 안쪽 루프만 종료합니다. 

## 예제
### 기본 사용 예제
```tcl
set sum 0
for {set i 1} {$i <= 10} {incr i} {
    if {$i == 5} {
        break
    }
    set sum [expr {$sum + $i}]
}
puts "Sum: $sum"  ;# 출력: Sum: 10
```

### 프로시저 내에서의 사용
```tcl
proc myProcedure {} {
    for {set i 0} {$i < 10} {incr i} {
        if {$i == 3} {
            break
        }
        puts $i
    }
}

myProcedure  ;# 출력: 0, 1, 2
```

## 설명
- `break`를 사용할 때 가장 주의해야 할 점은, 현재 실행 중인 루프만 종료된다는 것입니다. 중첩된 루프가 있을 경우, 가장 안쪽 루프만 영향을 받습니다.
- `break` 명령어는 조건문과 함께 사용하여 특정 조건을 만족했을 때만 루프를 중단하도록 설계할 수 있습니다.
- `break`는 반환값을 가지고 있지 않으며, 루프 또는 프로시저의 종료와 관련된 흐름 제어만 수행합니다.

## 한 줄 요약
Tcl의 `break` 명령어는 루프나 프로시저의 실행을 중단하고, 다음 단계로의 진행을 차단하는 데 사용됩니다.
