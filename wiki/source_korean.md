# Tcl의 "source" 명령어: 스크립트 파일을 불러오는 방법

## 개요
Tcl의 `source` 명령어는 외부 Tcl 스크립트 파일을 현재 스크립트 또는 인터프리터에 포함시키는 기능을 제공합니다. 이를 통해 코드의 재사용성을 높이고, 모듈화된 프로그래밍이 가능하게 합니다.

## 문서화
`source` 명령어는 Tcl 스크립트에서 다른 Tcl 파일의 내용을 읽어 실행할 수 있도록 해줍니다. 이 명령어는 다음과 같은 형식으로 사용됩니다:

```tcl
source 파일이름
```

### 목적
`source`의 주요 목적은 외부 스크립트 파일의 명령어를 현재의 Tcl 환경에서 사용할 수 있도록 하는 것입니다. 이를 통해 코드를 분리하여 관리할 수 있고, 공통 기능을 여러 스크립트에서 재사용할 수 있습니다.

### 사용법
- `파일이름`: 불러오고자 하는 Tcl 스크립트 파일의 경로를 지정합니다. 상대 경로 또는 절대 경로를 모두 사용할 수 있습니다.
- `source` 명령어는 파일을 읽고 실행한 후, 그 안에 정의된 프로시저 및 변수를 현재의 스코프에 추가합니다.

### 세부 사항
- 파일이 존재하지 않거나 읽을 수 없는 경우, Tcl은 에러를 발생시킵니다.
- `source` 명령어는 파일의 모든 내용을 즉시 실행하므로, 필요한 경우 조건문으로 감싸서 특정 상황에서만 실행되도록 할 수 있습니다.
- 스크립트 파일 내에서 다른 스크립트 파일을 다시 `source`할 수도 있습니다.

## 예제
1. 기본적인 사용 예제:
   ```tcl
   # main.tcl
   puts "메인 스크립트 시작"
   source sub_script.tcl
   puts "메인 스크립트 종료"
   ```

   ```tcl
   # sub_script.tcl
   puts "서브 스크립트 실행 중"
   ```

   이 예제에서는 `main.tcl`을 실행할 때 `sub_script.tcl`이 포함되어 "서브 스크립트 실행 중"이라는 메시지가 출력됩니다.

2. 조건부 실행 예제:
   ```tcl
   set runSubScript 1
   if {$runSubScript} {
       source sub_script.tcl
   }
   ```

## 설명
- **일반적인 함정**: `source` 명령어를 사용할 때, 파일 경로가 잘못되었거나 파일이 존재하지 않을 경우 에러가 발생합니다. 따라서 파일 경로를 정확히 지정해야 합니다.
- **스코프 문제**: `source`한 파일 내에서 정의된 변수나 프로시저는 현재 스크립트의 스코프에서 사용할 수 있습니다. 이로 인해 예상치 못한 변수 충돌이 발생할 수 있습니다.
- **실행 순서**: `source` 명령어는 파일 내의 모든 명령어를 순차적으로 실행하여, 실행 흐름에 영향을 줄 수 있습니다. 따라서 파일의 내용 순서를 고려해야 합니다.

## 한 줄 요약
Tcl의 `source` 명령어는 외부 Tcl 스크립트 파일을 현재 스크립트에 포함시켜 실행할 수 있게 해주는 강력한 도구입니다.
