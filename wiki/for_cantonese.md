# Tcl 中的 “for” 迴圈語法

## 概述
在 Tcl 程式語言中，`for` 迴圈是一個用於重複執行一段代碼的結構，特別適合用於已知重複次數的情況。它可以簡化代碼並提高可讀性。

## 文檔
### 目的
`for` 迴圈的主要目的是允許開發者根據指定的初始值、條件和增量來重複執行一段代碼。

### 語法
```tcl
for {初始化} {條件} {增量} {
    # 代碼區塊
}
```

- **初始化**：在迴圈開始前執行的命令，通常用於設定迴圈變量。
- **條件**：每次迴圈執行前評估的條件，當條件為真時，迴圈繼續執行。
- **增量**：在每次迴圈執行結束後執行的命令，通常用於更新迴圈變量的值。

### 使用方式
`for` 迴圈的基本使用方式如上所示，開發者可以根據需要在代碼區塊中執行任何 Tcl 命令。

## 範例
以下是一些基本的使用範例：

### 範例 1：打印數字
```tcl
for {set i 0} {$i < 5} {incr i} {
    puts $i
}
```
這段代碼會輸出 0 到 4 的數字。

### 範例 2：計算總和
```tcl
set sum 0
for {set i 1} {$i <= 10} {incr i} {
    set sum [expr {$sum + $i}]
}
puts "總和是: $sum"
```
這段代碼會計算從 1 到 10 的總和並顯示結果。

## 解釋
在使用 `for` 迴圈時，開發者應注意以下幾點：

- **條件評估**：確保條件能在某個時候變為假，以避免無限迴圈。
- **變量範圍**：`for` 迴圈中使用的變量在迴圈外不可見，若需要在迴圈外使用，請使用全局變量或在迴圈外進行定義。
- **增量操作**：確保增量操作正確，以免導致邏輯錯誤。

## 總結
Tcl 的 `for` 迴圈是強大的控制結構，適合用於需要重複執行特定代碼段的情境，並能提高程式的可讀性與維護性。
