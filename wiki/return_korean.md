# Tcl의 "return" 명령어: 함수에서 값 반환하기

## 개요
Tcl에서 "return" 명령어는 프로시저나 함수의 실행을 종료하고, 호출한 위치로 값을 반환하는 데 사용됩니다. 이 명령어는 함수의 결과를 클라이언트 코드에 전달하는 중요한 역할을 합니다.

## 문서화
### 목적
"return" 명령어는 Tcl 프로시저 내에서 값을 반환하고, 프로시저의 실행을 종료하는 기능을 제공합니다. 이를 통해 다양한 연산의 결과를 호출한 코드로 전달할 수 있어 프로시저의 재사용성을 높입니다.

### 사용법
```tcl
return ?value?
```

- `value`: 반환할 값. 문자열, 숫자 또는 리스트 등 다양한 데이터 유형이 가능합니다. 이 매개변수는 선택 사항이며, 제공하지 않을 경우 빈 문자열이 반환됩니다.

### 세부사항
- "return" 명령어는 프로시저 내에서만 사용될 수 있습니다. 프로시저 외부에서 호출하면 에러가 발생합니다.
- 반환된 값은 프로시저를 호출한 코드에서 사용할 수 있으며, 여러 개의 값을 반환할 수도 있습니다.
- 반환된 값이 없으면 기본적으로 빈 문자열이 반환됩니다.

## 예제
### 기본 사용 예
```tcl
proc add {a b} {
    return [expr {$a + $b}]
}

set result [add 5 10]
puts $result  ;# 출력: 15
```

### 여러 값 반환하기
```tcl
proc getCoordinates {} {
    return 10 20
}

set coords [getCoordinates]
puts $coords  ;# 출력: 10 20
```

## 설명
"return" 명령어를 사용할 때 주의해야 할 몇 가지 사항이 있습니다:

- **프로시저 외부에서 사용 금지**: "return"은 반드시 프로시저 내에서만 사용해야 하며, 외부에서 호출하면 오류가 발생합니다.
- **여러 값 반환**: 반환할 값이 여러 개인 경우 공백으로 구분하여 반환할 수 있습니다. 호출한 측에서는 리스트 형태로 받을 수 있습니다.
- **에러 처리**: 프로시저의 실행 도중 오류가 발생하면, "return" 명령어는 실행되지 않거나, 명시적으로 오류를 발생시킬 수 있습니다. 이 점을 고려하여 에러 처리를 적절히 구현해야 합니다.

## 한 줄 요약
Tcl의 "return" 명령어는 프로시저에서 값을 반환하고 실행을 종료하는 데 사용됩니다.
